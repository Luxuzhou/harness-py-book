# 安全说明

## 报告安全问题

请不要公开提交可被利用的未修复漏洞、密钥或个人信息。优先使用 GitHub 仓库的 Security Advisory 私密报告功能；如该功能暂不可用，请联系仓库维护者并仅提供复现所需的最少信息。

## 教学性缺陷

第 9—11 章的案例会故意保留待 Agent 识别和修复的问题，例如 SQL 拼接、PII 输出、权限边界和遗留代码坏味道。这些内容是隔离的教学材料，不应直接部署到生产环境。

`cases/data_compliance/target_service/sample_data/` 中的数据为合成数据。即使如此，运行实验时也应按真实敏感数据的标准保护日志、导出文件和 API 凭据。

## 凭据保护

- 只在本地 `.env` 中保存 API Key；该文件已被 `.gitignore` 排除。
- 提交前检查 `git status` 和暂存区，不要上传 session、日志、导出文件或构建产物。
- 如果密钥曾进入 Git 历史，请立即在供应商控制台撤销并轮换，不能只删除工作区文件。
